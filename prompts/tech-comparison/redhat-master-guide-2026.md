# Role
あなたは Red Hat 認定トレーニングのシニア・エバリュエーター兼、試験対策スペシャリストです。

# Task
RHEL、OpenShift、Ansible、および RHLS（Learning Subscription）を網羅し、さらに「実機試験を突破するための具体的テクニック」を含めた、2026年最新の Red Hat 認定資格完全攻略スライド構成案を作成してください。

# ターゲット層
- 初学者からスペシャリスト、アーキテクト（RHCA）を目指すエンジニア
- 実機試験の形式に不安があり、具体的な「受験のコツ」を知りたい人
- 自宅受験（Remote Exam）の準備方法やトラブル対策を知りたい人

# 反映すべき重要データとルール (2026年3月時点)

1. **コードの読み方と難易度**
   - **接頭辞の意味**: `RH` = RHEL系、`DO` = OpenShift/コンテナ系、`AU` = Ansible系、`EX` = 試験(Exam)。
   - **数字の法則**: 百の位がレベル（難易度）を示す（100番台：入門、200番台：中級、300番台：上級）。
   - **対応関係**: 原則「トレーニングで学び、同じ番号の試験を受ける」（例: DO156を履修 → EX156を受験）。

2. **認定の基本ルールと称号**
   - **1試験 = 1認定**: 基本的に1つの試験（EX）に合格すれば、その分野のスペシャリストとして認定される。
   - **称号の取得条件**:
     - **RHCSA**: EX200 に合格。
     - **RHCE**: 「有効な RHCSA」を保有した状態で、EX294（Ansible）に合格。
     - **RHCA (最高峰)**: 「有効な RHCE」を保有した状態で、任意の5つのスペシャリスト認定（EX）を取得。

3. **推奨学習パス（RHEL）**
   - 初学者: RH124 + RH134 → EX200 (RHCSA)。
   - 経験者（速習）: **RH199** → EX200 (RHCSA)。

4. **実機試験突破の「5つの鉄則」**
   - **Copy & Paste**: スペルミス（タイポ）は致命傷。問題文からのコピペを徹底する。
   - **ドキュメント活用**: 試験環境内の公式ドキュメント（man, help等）を素早く引く練習をする。
   - **解く順番**: 依存関係を確認し、確実に解ける問題から着手してスコアを稼ぐ。
   - **言語設定**: 日本語・英語の切り替えを活用し、問題のニュアンスを正確に把握する。
   - **自宅受験の備え**: 専用USB起動メモリの事前作成、OS起動失敗時のトラブル対応手順を把握しておく。

5. **RHLS (Learning Subscription) のプラン比較**
   - **Course**: 1コース限定（1試験＋1リテイク）。特定資格に集中。
   - **Standard**: 全カタログ自習（5試験＋5リテイク）。幅幅広く学びたい人向け。
   - **Premium**: Standard ＋ 全コース最大2回ずつのライブ講義。確実に合格したい人向け。
