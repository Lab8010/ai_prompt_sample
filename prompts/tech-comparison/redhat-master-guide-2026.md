# 反映すべき重要データとルール (2026年3月時点)

1. **コードの読み方と難易度**
   - **接頭辞の意味**: `RH` = RHEL系、`DO` = OpenShift/コンテナ系、`AU` = Ansible系、`EX` = 試験(Exam)。
   - **数字の法則**: 百の位がレベル（難易度）を示す。
   - **対応関係**: 原則「トレーニングで学び、同じ番号の試験を受ける」（例: DO156 → EX156）。

2. **認定の基本ルールと称号**
   - **1試験 = 1認定**: 得意な製品から受験可能。
   - **称号の取得条件**:
     - **RHCSA**: EX200 に合格。
     - **RHCE**: 有効な RHCSA を保有した状態で、EX294 に合格。
     - **RHCA**: 有効な RHCE を保有した状態で、任意の5つの認定（EX）を取得。

3. **実機試験突破の「5つの鉄則」**
   - **Copy & Paste**: スペルミス防止のため問題文からのコピペを徹底。
   - **ドキュメント活用**: 試験環境内の man や help を引きこなす。
   - **解く順番**: 確実に解ける問題から着手してスコアを稼ぐ。

4. **自宅受験（Remote Exam）のデスク環境要件**
   - **外部モニター禁止**: ラップトップの内蔵画面のみ。外部モニターは取り外すかカバーをする。
   - **デスク上の整理**: スマホ、時計、メモ、ペン等は一切禁止。**ただし、飲み物は置いてOK。**
   - **カメラ配置**: 部屋全体と机の上が確認できる配置。
   - **トラブル対策**: 専用USB起動メモリの事前作成、OS起動失敗時のFAQ確認。

5. **最強のシミュレーター「Exam Experience」**
   - **概要**: RHLS利用者が利用可能な、本番と全く同じプロセスで受けられる模擬試験。
   - **メリット**: 難易度・雰囲気ともに本番を忠実に再現。事前に体験することで当日の焦りを解消できる。

6. **RHLS (Learning Subscription) のプラン比較**
   - **Course**: 1資格集中。**Standard / Premium**: 全カタログ＋試験5回＋Exam Experience利用可。
