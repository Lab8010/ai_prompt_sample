1
概要
Red Hat 認定資格とは？

Red Hat 認定は、エンタープライズ Linux・クラウド・自動化の領域で世界的に認められた IT 資格体系です。選択式問題ではなく、実機操作による実技試験のみで合否が判定される点が最大の特徴です。

試験形式
実機操作のみ
選択式・穴埋めなし。実際のシステムを動かして採点
対象技術
RHEL / OpenShift / Ansible
Linux 基盤からコンテナ・自動化まで幅広くカバー
受験方法
会場受験 / 自宅受験
Remote Exam により自宅の PC から受験可能

なぜ価値があるのか：実機で動かして初めて合格できる形式のため、「知識だけ」の暗記では通用しません。取得者はその技術を実際に使いこなせる証明として、採用・昇進・案件獲得の場面で評価されます。

「Red Hat の資格は聞いたことがあるけれど、どんな試験なの？」という疑問からスタートしましょう。

最も伝えるべき点は「実技試験のみ」という事実です。TOEIC や情報処理試験のような選択式・記述式は一切なく、制限時間内に実際の Linux 環境上でコマンドを打ち、設定を完成させ、サービスを起動する——その結果だけで合否が決まります。

だからこそ、「試験に合格している = 実際にできる人」という市場の信頼が生まれています。採用担当者・プロジェクトマネージャーが RHCSA・RHCE という文字を履歴書で見たとき、即戦力として評価するのはそのためです。

2
体系理解
コース・試験コードの読み方

コードには意味があります。接頭辞でドメインが、百の位の数字で難易度がわかります。トレーニングと試験は同じ番号で対応しているため、学習パスが直感的に設計できます。

接頭辞 ドメイン 代表的なコード 内容
RH RHEL 系 RH124, RH134, RH199 Red Hat Enterprise Linux の管理・運用
DO OpenShift / コンテナ系 DO180, DO280, DO156 Kubernetes ベースのコンテナプラットフォーム
AU Ansible 系 AU294 Ansible による IT 自動化・構成管理
EX 試験 (Exam) EX200, EX294, EX188 実技試験。対応トレーニングと番号が一致

百の位 = 難易度レベル： 1xx は入門・基礎、2xx は中級、4xx 以上は上級・専門領域。例）DO180（入門）→ DO280（中級）と段階的に学べます。

コードの規則性を理解すると、カタログが一気に読めるようになります。「DO280 を受けたい」と思ったら、対応する試験は「EX280」です。番号が同じなのでシンプルです。

よく混乱するポイントは 「RH と AU の違い」。RH は RHEL（Linux そのもの）のトレーニング、AU は Ansible（自動化ツール）のトレーニングです。最終的には両方学ぶことになる場合が多いですが、最初は別ドメインとして区別して覚えましょう。

「EX は全部試験コード」と覚えておくと、カタログを眺めたときに「これは学ぶコース」「これは受験するもの」と即座に判別できます。

「有効な」認定について： 各認定には有効期限があります。RHCSA が失効すると RHCE の要件を満たせなくなるため、定期的な更新（再受験）が必要です。学習を継続するインセンティブにもなっています。

「どこから始めればいいですか？」という質問への答えは常に RHCSA（EX200）から です。これは前提資格ではなく、ここが全ての出発点です。

RHCE は EX294（Ansible）の試験ですが、受験時点で有効な RHCSA が必要です。つまり RHCSA が「門番」の役割を果たしています。順番を飛ばすことはできません。

RHCA は最上位称号ですが、「任意の 5 認定」という自由度があります。OpenShift を専門にするもよし、セキュリティ系を選ぶもよし。自分のキャリア方向に合わせて設計できる点が魅力です。

受講者にはこう伝えるといいでしょう：「まず EX200 を取ることだけを考えてください。その先は自然に見えてきます。」

4
学習パス — 基礎〜中級
RHEL と Ansible で土台を作る

Linux とその自動化は、すべての上位認定の土台です。RH 系コースで Linux の実務運用を身につけ、AU 系コースで Ansible による自動化を習得することで RHCSA → RHCE を目指せます。

RH RHEL 系
RH124 — RHEL 基礎 I
CLI 操作、ファイル管理、ユーザー管理の基本
RH134 — RHEL 基礎 II
ネットワーク、ストレージ、SELinux、サービス管理
RH199 — RHCSA 速習
RH124+RH134 の内容を短期集中で習得
AU Ansible 系
AU294 — Ansible 自動化
Playbook 作成、ロール管理、Linux 自動化の実践
対応試験：EX294 に合格すると RHCE 取得

時間が取れない場合の選択肢： RH199（RHCSA Rapid Track）は RH124 と RH134 を圧縮した短期集中コースです。ある程度 Linux 経験がある方に特に有効です。

RHCSA を目指す方には、Linux 経験がほぼない場合は RH124 → RH134 の順で受講することを推奨してください。ある程度経験がある場合は RH199 で時間を圧縮できます。

Ansible（AU294）は「Linux を自動化する」というコンセプトなので、まず Linux そのものをしっかり理解してからが効果的です。RHCSA 取得後に AU294 に進む流れが最も学習効率が高いです。

「Ansible って何ですか？」と聞かれたら：「多数のサーバーに同じ設定を一括で自動適用するための言語・ツールです。100 台のサーバーに設定を手動で入れる代わりに、1 つの設定ファイルを実行するだけで完了します」と説明するとイメージしやすいです。

学習パス — 最先端技術
OpenShift とコンテナで最前線へ

クラウドネイティブ時代のインフラエンジニアに求められる技術が DO 系コースに集まっています。Kubernetes ベースの OpenShift を扱う能力は、現在の市場でもっとも需要が高いスキルセットの一つです。

DO180
OpenShift 入門
コンテナとは何か。OpenShift 上での基本的なアプリケーションデプロイを習得
DO280
OCP 管理
OpenShift クラスター自体の設定・管理・セキュリティを担当するエンジニア向け
DO156
Virtualization
VM とコンテナを同一プラットフォームで管理するハイブリッドインフラを学ぶ
RHCA への貢献
DO 系の各試験（EX180、EX280、EX156 等）に合格することで RHCA に必要な「5 認定」のカウントに含まれます。OpenShift を専門とするアーキテクトを目指す場合、DO 系を中心に組み立てることが可能です。

「コンテナ」「Kubernetes」「OpenShift」という言葉に馴染みがない受講者がいる場合、次のように説明できます：「従来はアプリを動かすために専用のサーバーを用意していましたが、コンテナ技術を使えば一台のサーバー上で多数のアプリを安全に分離して動かせます。OpenShift はそれを大規模に管理するための基盤です」

DO180 は RHCSA 取得後に並行して進めることもできますが、Linux の基礎知識が前提になります。完全な初心者には RHEL 系から始めることを推奨します。

「現場で使われているか？」という問いに対しては： 現在、金融・製造・通信・官公庁を含む多くの大企業が OpenShift 上でシステムを稼働させており、OpenShift を扱えるエンジニアへの需要は継続して高い水準にあります。

6
試験の特徴
実機試験だからこそ、本物の実力が証明できる

Red Hat 試験は「実機操作のみ」で合否が決まる実技試験です。他の IT 資格と何が違うのか、具体的な形式と合格のためのポイントを整理します。

一般的な IT 試験との違い
  • ✗選択式問題は一切なし
  • ✗穴埋め・記述問題もなし
  • ✓実際の RHEL 環境でコマンドを打ち、設定を完成させる
  • ✓採点は自動スクリプト。「動いているか」だけが評価基準
合格への 3 つの実践的ポイント
01 Copy & Paste を徹底：問題文からコマンドをコピーしてスペルミスを防ぐ
02 man / --help を活用：試験環境内のドキュメントは使用可
03 確実に解ける問題から着手：スコアを積み上げて心理的余裕を確保

実技試験の価値： 「知っている」と「できる」は違います。実機で合格した認定者は、現場で即戦力として機能します。採用・昇進の場面でこの違いが評価されます。

「実技試験って難しくないですか？」という不安に対しては：「難しいのは事実ですが、裏を返せばしっかり練習した人が必ず合格できる試験です。暗記だけでは突破できない分、練習した分だけ結果に直結します」

man コマンドの使用について： 試験中に Linux のマニュアルを参照することは、カンニングではなく「許可されているリソースの活用」です。実務でも同様に使うスキルなので、試験でも積極的に使うよう伝えましょう。

Copy & Paste については「なぜ重要か」を強調すると納得度が上がります：試験中に `configure` を `confgure` とミスタイプするだけでコマンドが失敗し、貴重な時間を失います。実務でも正確にコマンドを入力することは基本の基本です。

7
自宅受験
Remote Exam — 自宅受験の環境要件

Red Hat は会場受験に加え、自宅の PC から受験できる Remote Exam を提供しています。試験官がカメラ越しに環境を確認するため、事前の準備が重要です。

デスク・機材のルール
  • ✗外部モニター禁止：ラップトップの内蔵画面のみ。取り外しかカバーが必要
  • ✗スマートフォン・腕時計・メモ帳・ペン：すべてデスク上に置けない
  • ✓飲み物は置いてもOK。長丁場の試験、水分補給は許可
  • ✓部屋全体とデスク上が確認できるカメラ配置が必要
技術的な事前準備
  • ✓Red Hat 提供の 専用 USB 起動メモリ を事前に作成
  • ✓OS 起動失敗時に備え、公式 FAQ を事前確認
  • ✓ネットワーク接続の安定性を事前に確認・テスト

当日に慌てないために： 技術的なトラブル（USB 起動失敗、カメラ認識不具合など）は事前準備で防げるものがほとんどです。受験申し込み後、早めに環境構築を済ませておくことを強くお勧めします。

「飲み物 OK」は意外と喜ばれるポイントです。試験時間は 2〜3 時間あるため、水分補給の可否は集中力に直結します。緊張している受験者に対して「飲み物だけは持ち込めます」と伝えると、少し安心感が生まれます。

外部モニター禁止については「なぜ？」と聞かれることがあります。試験の公平性と不正防止のため、画面が複数あると試験以外の情報を参照している可能性を排除できないためです。

USB 起動メモリの重要性： Remote Exam では PC を通常の OS ではなく、Red Hat が提供する専用の環境から起動して試験を受けます。このメモリが作成できていないと、試験当日に起動すらできない事態になります。必ず早めに準備するよう伝えましょう。

8
RHLS 特典
Exam Experience — 本番と同じ環境で練習できる

Exam Experience は、RHLS（Red Hat Learning Subscription）のサブスクリプション利用者が使える模擬試験サービスです。本番試験と全く同じプロセス・環境で受験できるため、当日の不安を大幅に軽減できます。

本番再現度
高い
試験監督チェック、インターフェース、制限時間まで本番と同一のプロセス
難易度
本番同等
実際の試験難易度と雰囲気を事前体験できる唯一の手段
利用条件
RHLS Standard /
Premium のみ
Course プランには含まれないため、プラン選択時に確認を

「初めて」の不安を事前に解消： 試験に臨む前に「本番と同じ感覚」を一度経験しておくことで、当日の焦り・操作の迷いを大幅に減らせます。特に Remote Exam が初めての方にとっては非常に有効な準備手段です。

「本番に近い環境で練習できる」というのは、スポーツでいえば「同じスタジアムで練習試合ができる」ようなものです。技術的な準備が十分でも、試験の独特な緊張感・インターフェース・時間配分に慣れていないと本番で実力を発揮できないことがあります。

Exam Experience を使った受験者の多くが「当日は 2 回目だという感覚だった」と報告しています。「初めての感覚」を模擬試験で消化してから本番に臨むことができるのは、RHLS Standard/Premium の最大のメリットの一つです。

RHLS Course プランとの違いとして、この点を強調することで Standard/Premium の投資対効果をより具体的に説明できます。

Red Hat Learning Subscription（RHLS）は、Red Hat 認定資格の学習を体系的にサポートするサブスクリプションです。目的と予算に応じて 3 つのプランから選べます。

判断の目安： RHCSA だけを短期で取りたい場合は Course。RHCE・RHCA まで見据えたキャリア設計をしたい場合は Standard / Premium が費用対効果の高い選択です。Exam Experience を活用できる点も大きな違いです。

プラン選択で最もよくある質問は「Course と Standard どちらがいいですか？」です。答え方の目安：「1 つの資格だけ取る予定で、あとはその資格だけで十分という方には Course で十分です。ただし、RHCSA を取ったあとに RHCE も目指したい、OpenShift も勉強したいと少しでも思うなら、最初から Standard を選ぶ方がトータルのコストは低くなります」

試験チケット 5 回分の価値も具体的に説明できます：試験を 1 回個別に申し込むコストと比較すると、Standard プランで含まれる 5 回分のチケットだけで、サブスクリプション費用のかなりの部分を回収できる計算になります。

まとめのメッセージとして：「今日学んだこと——試験体系、称号、学習パス、そして Exam Experience という練習環境——これらを使えば、Red Hat 認定は決して遠い目標ではありません。まず EX200（RHCSA）という最初の一歩を踏み出す準備を、ぜひ今日から始めてみてください。」
